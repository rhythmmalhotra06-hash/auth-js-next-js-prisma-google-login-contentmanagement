For Vishen: one view, end to end

Every request Vishen makes — from the moment he asks for it, through production, to where it got posted and how it performed — in a single place he and the team can both see.

🎬 Request → Produced → Published → Performance 👀 One thread per request 📊 Live numbers
1 The gap today

Vishen can't see his own content flow

His requests live across Jira and several Airtable bases. Once something is produced, there's no single answer to two questions he keeps asking:

?
“Who edited this, and where is it now?” — status is scattered; nothing ties a request to where it got posted.
?
“How did it perform?” — published links and performance numbers aren't captured against the request at all.
?
“What did we publish last year?” — no filterable, shareable record of what shipped for him.
2 The solution

One thread, four stages

A view built for Vishen and the team: the team sees what's ready to publish; Vishen sees it go live with performance. Every item flows through the same four stages, colour-coded end to end.

In production
Being made now, by whom.
Awaiting sign-off
Needs Vishen's review.
Ready to publish
Approved — team posts it.
Published
Live link + performance.
▶ Open the live prototype ✎ Manual-entry version
3 How performance gets in

Free, open-source, and live now

✎
Start manually — todayThe moment a post goes live, the team drops the link and logs the numbers. Vishen sees performance immediately, no tooling required.
⚙️
Then automate — with an open-source tool (Postiz)Self-hosted, free, no per-seat cost. It pulls the numbers automatically from the connected accounts. Manual and automatic use the same screen — so nothing gets rebuilt when we switch on automation.
⚠️
Heads up on the metric: Instagram retired “impressions” in April 2025 — the number everyone now reports is Views + Engagement rate. We should agree to report those.
