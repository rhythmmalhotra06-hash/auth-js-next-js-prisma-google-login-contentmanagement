# Performance Loop

> Part of [Mindvalley Content Production & Management System](../content-production-management.md)

> **Deferred to Phase 2.** The Phase 1 ship gate ends at Published; this epic closes the lifecycle to Performance Tracked afterward.

## Purpose

Close the differentiator loop: wire the `performance` table to published assets and surface metrics in the stakeholder view so "who edited this AND how did it perform" is answered in one place. Phased internally — manual metric entry first, then automated connectors (Clarisights, Amplitude, Ahrefs, all already connected at Mindvalley). Whether metrics live on the Prio table vs Asset Library is an open team decision (currently modeled library-side in `schema.sql`).
